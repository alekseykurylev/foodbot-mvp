# Структура `lib`

- `bots/` - рантайм ботов, команды и обработчики вебхуков.
- `cms/` - клиенты Payload CMS и инфраструктура, специфичная для CMS.
- `domain/` - бизнес-операции, сгруппированные по доменным моделям.
- `integrations/` - клиенты внешних сервисов и обертки над API.
- `mini-app/` - авторизация Mini App, launch data, client hooks и общие типы сессии.
- `helpers/` - небольшие универсальные helpers без привязки к домену.

Новый код лучше добавлять в самую узкую папку, которая отвечает за эту область. Route handlers стоит держать тонкими, а переиспользуемое поведение выносить в `domain/`, `integrations/` или соответствующую feature-папку.

Feature-specific hooks храните рядом с feature-кодом. Верхнеуровневую папку `hooks/` стоит заводить только для общих UI/browser hooks, которые не принадлежат конкретной области.
